# Project workflow

处理本项目请求时，先阅读项目内相关文档和最近变更记录，再动手修改。涉及外部 API、线上部署、仓库地址、云端路径或产品资料时，先核实现有文档、代码引用和线上状态；资料可能过期时，优先查官方或当前线上结果。

## Images

- 不在对话里重复粘贴大段图片 base64 或完整二进制响应。
- 生成或处理图片后，保存为项目文件，只在回复里给出路径、尺寸、大小和验证结论。
- 运行态图片优先使用轻量 WebP；旧 PNG/JPG/PPT/Keynote 源文件不要作为部署资源上传。
- 新增透明背景图后，检查 alpha 通道、四角透明度、页面引用和热点位置。

## Deploy

- 修改完成后先提交并推送 GitHub。
- GitHub 推送成功后，才同步服务器。
- 服务器同步前先备份线上目录。
- 不读取、展示或提交任何私钥、SecretKey、密码或完整环境配置。
